Step 0配置 Python 环境

Python 是数据分析的基础语言。我们使用 Miniforge（conda 的轻量社区版）来管理 Python 环境，避免版本冲突。

  1. 访问 conda-forge.org/download，下载对应芯片的安装包：
    • Apple Silicon（M1/M2/M3/M4）：Miniforge3-MacOSX-arm64.sh
    • Intel Mac：Miniforge3-MacOSX-x86_64.sh
    不知道芯片类型？点击左上角苹果菜单 → "关于本机"查看。
  2. 打开终端（启动台搜索"终端"，或按 Cmd+Space 输入 Terminal），运行： 
    # Apple Silicon
bash ~/Downloads/Miniforge3-MacOSX-arm64.sh

# Intel Mac
bash ~/Downloads/Miniforge3-MacOSX-x86_64.sh
    按提示输入 yes 并按 Enter，一路确认。
  3. 安装完成后，关闭并重新打开终端。若提示符前出现 (base)，说明 conda 已成功激活。
  4. 创建专用数据科学虚拟环境： 
    conda create -n datasci python=3.10
conda activate datasci
conda install pandas numpy scipy scikit-learn matplotlib seaborn jupyter
✅ 验证：在终端运行 python --version，显示 Python 3.10.x 即成功。
  1. 访问 conda-forge.org/download，下载 Miniforge3-Windows-x86_64.exe。
  2. 双击安装包，按图形界面指引安装：
    • 建议勾选 "Add Miniforge3 to PATH environment variable"（方便命令行使用）
    • 其余选项保持默认
  3. 安装完成后，从开始菜单搜索并打开 "Miniforge Prompt"（注意：不是 PowerShell 或 CMD）。
  4. 在 Miniforge Prompt 中创建环境：

✅ 验证：在 Miniforge Prompt 运行 python --version，显示 Python 3.10.x 即成功。

Step 1安装编辑器（VS Code）

VS Code 是 Claude Code 的主要工作台，你通过它与 AI 对话、编写代码、管理项目文件。

备选方案：Trae IDE（字节跳动出品，内置 AI，界面与 VS Code 相似）。两者选一即可，本指南以 VS Code 为主。
  1. 访问 code.visualstudio.com，下载 macOS 版（.dmg）。
  2. 打开 .dmg，将 Visual Studio Code 拖拽到"应用程序"文件夹。
  3. 首次打开时，若系统提示"来自互联网的应用"，点击"打开"即可。
  4. 推荐安装命令行工具：打开 VS Code，按 Cmd+Shift+P，输入 "Shell Command: Install 'code' command in PATH"，回车。之后可在终端输入 code . 用 VS Code 打开当前文件夹。
  5. 打开 VS Code，点击左侧 Extensions（扩展） 图标，搜索并安装：
    • Python（Microsoft 出品）
    • Jupyter（Microsoft 出品）

测试 Jupyter 环境

print("Python 环境正常！")

✅ 若看到描述性统计数据，环境配置成功。

1.2 登录 VS Code（可选，推荐）

登录 VS Code 账号可以同步你的设置、插件和使用记录到云端，换电脑后自动恢复。

  1. 打开 VS Code，点击左下角 账户图标（人形图标）→ 选择 "登录以同步设置"（Sign in to Sync Settings）。
  2. 选择登录方式：

    • Google 账号

登录的好处：你之前在其他电脑上的 VS Code 设置、安装的插件、Claude Code 使用记录等，都会自动同步到当前电脑。如果不登录，以上内容仅保存在本地。

Step 2安装 Node.js

Claude Code 需要 Node.js 运行环境。Node.js 相当于 Claude Code 与你电脑之间的"翻译官"。

方式一（推荐）：通过 Homebrew 安装

若尚未安装 Homebrew，先在终端运行（仅需一次）：

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

然后安装 Node.js：

brew install node

方式二：直接下载安装包

访问 nodejs.org/en/download，下载 macOS Installer（.pkg），双击安装。

✅ 验证：终端运行 node -v 和 npm -v，均显示版本号即成功（如 v22.x.x）。
  1. 访问 nodejs.org/en/download，下载 Windows Installer (.msi)（选 LTS 版本）。
  2. 双击安装，按默认选项一路 Next 即可。
  3. 安装完成后，打开 PowerShell（按 Win 键，输入 PowerShell，回车），验证： 
    node -v
npm -v
常见报错：脚本执行策略限制
若看到 "无法加载文件，因为在此系统上禁止运行脚本"，执行以下修复（约1分钟）：
  1. 右键开始菜单 → "Windows PowerShell（管理员）"（或"终端（管理员）"）
  2. 运行： 
    Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope LocalMachine
    提示确认时按 Y + 回车
  3. 重新打开普通 PowerShell，再次运行 npm -v
若仍报错：Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
✅ 验证：node -v 和 npm -v 均显示版本号即成功。

Step 3安装 Git

Git 是版本管理工具，Claude Code 需要它来下载依赖和管理项目文件。

打开终端，运行：

git --version
  • 若显示版本号（如 git version 2.39.0），说明已预装，跳过本步骤。
  • 若提示安装，点击系统弹窗中的"安装"按钮（macOS 会自动安装 Xcode Command Line Tools，其中包含 Git）。

如需最新版，可通过 Homebrew 安装（若尚未安装 Homebrew，见下文）：

brew install git

初次配置（推荐）

设置用户名和邮箱，Git 提交记录会带上这些信息：

git config --global user.name "你的姓名"
git config --global user.email "你的邮箱@example.com"
✅ 验证：终端运行 git --version，显示版本号即成功。
  1. 访问 Git 官网：git-scm.com/downloads/win，点击 "Click here to download"，下载最新版 .exe 安装包。
    官网会自动匹配你的系统版本（64 位或 32 位），无需手动选择。
  2. 双击安装包，按图形界面指引安装。以下是推荐选项（其余保持默认即可）：
    • "Adjusting your PATH environment" → 选 "Git from the command line and also from 3rd-party software"
    • "Choosing the default editor" → 建议选 "Use Visual Studio Code as Git's default editor"
    • "Configuring the line ending conversions" → 选 "Checkout Windows-style, commit Unix-style line endings"
    • 其余页面一路 Next（保持默认）即可
  3. 安装完成后，PowerShell 验证： 
    git --version

初次配置（推荐）

git config --global user.name "你的姓名"
git config --global user.email "你的邮箱@example.com"
✅ 显示版本号（如 git version 2.44.0.windows.1）即成功。

Step 4安装 Claude Code

Claude Code 是本套工作流的核心 AI 引擎，通过命令行和 VS Code 插件与你互动。

打开终端，运行：

sudo npm install -g @anthropic-ai/claude-code

输入系统密码（输入时字符不显示，属正常现象），按回车。等待安装完成（约1-3分钟）。

✅ 验证：终端运行 claude，若弹出欢迎界面或登录提示，安装成功。
  1. 右键开始菜单 → 选择 "Windows PowerShell（管理员）" 或 "终端（管理员）"。
    为什么需要管理员权限？npm 全局安装（-g 参数）需要写入系统目录，普通权限可能因权限不足而失败。
  2. 在管理员 PowerShell 中运行： 
    npm install -g @anthropic-ai/claude-code
    等待安装完成（约 1–3 分钟）。
下载缓慢？使用国内镜像： 
npm install -g @anthropic-ai/claude-code --registry=https://registry.npmmirror.com

验证安装

关闭管理员窗口，打开普通 PowerShell，运行：

claude --version
✅ 验证：显示版本号（如 v1.0.x）或运行 claude 弹出欢迎界面/登录提示，即安装成功。

网络代理配置（中国大陆用户必看）

由于 Anthropic 服务器在境外，国内直连时常出现以下报错：

Unable to connect to Anthropic services
Failed to connect to api.anthropic.com: ERR_BAD_REQUEST
Please check your internet connection and network settings.

解决方法：在运行 claude 之前，设置代理环境变量。你需要有一个可用的 VPN 或代理工具，并确认本地监听端口（常见：7890、1080、8080，以 VPN 客户端显示为准）。

临时设置（每次使用时手动执行）

export HTTP_PROXY="http://127.0.0.1:7890"
export HTTPS_PROXY="http://127.0.0.1:7890"
claude

永久设置（推荐）

将代理配置写入 shell 配置文件，每次打开终端自动生效：

echo 'export HTTP_PROXY="http://127.0.0.1:7890"' >> ~/.zshrc
echo 'export HTTPS_PROXY="http://127.0.0.1:7890"' >> ~/.zshrc
source ~/.zshrc
Shell 类型注意：macOS 默认使用 zsh（配置文件 ~/.zshrc）。如果你用的是 bash（旧系统），请改为 ~/.bash_profile。不确定的话运行 echo $SHELL 查看。

临时设置（每次使用时手动执行）

$env:HTTP_PROXY="http://127.0.0.1:7890"
$env:HTTPS_PROXY="http://127.0.0.1:7890"
claude

永久设置（推荐）

以管理员身份运行 PowerShell，执行：

[System.Environment]::SetEnvironmentVariable("HTTP_PROXY", "http://127.0.0.1:7890", "User")
[System.Environment]::SetEnvironmentVariable("HTTPS_PROXY", "http://127.0.0.1:7890", "User")

重启终端后永久生效。

端口号确认：不同 VPN 的本地代理端口不同。以 Clash 系列为例通常是 7890，V2Ray 通常是 10808，SSR 通常是 1080。请在你的 VPN 客户端设置中确认正确的端口号，否则代理不会生效。

Step 4.5替代与补充：Google Antigravity 与 Gemini 配置

Google Antigravity 是谷歌推出的一款面向开发者、以 AI Agent（智能体）为核心的下一代 AI 辅助编程与开发平台。

它不再只是像传统编辑器侧边栏里塞一个简单的 AI 聊天框，而是让 AI Agent 具备高度自主性：

  • 自主任务编排：你给它一个高层指令（比如“帮我写一个登录页面并对接数据库”），AI Agent 能够自主创建计划、编写代码、在终端运行服务，甚至自己打开浏览器进行测试和验证。
  • 多 Agent 并行：它的 Manager 视图像一个中央指挥控制中心，允许你同时派发多个不同任务给多个 AI 子智能体，让它们在后台异步、并行地协作完成工作。
  • 提供桌面端与 IDE：它拥有独立的桌面应用（Antigravity 2.0），同时也提供专门的 IDE（基于 VS Code 深度定制改造）以及 CLI 命令行工具。
  • Artifacts（交付物）机制：为了建立信任，AI 在执行任务时会生成清晰的任务列表、实现计划、甚至浏览器测试的截图和录屏，而不是让你去一行行对枯燥的运行日志。

简单来说，它是谷歌用来对标 Cursor、Windsurf 以及 GitHub Copilot Workspace 的重量级 AI 编程生态。

下载 Antigravity 2.0 与 Antigravity IDE

  1. 访问 Google Antigravity 官网，查找最新版本下载入口。
  2. 下载并安装 Antigravity 2.0 桌面客户端，该客户端适用于 macOS 与 Windows。
  3. 如果你希望在一个代码编辑器内使用 Antigravity，选择下载并安装 Antigravity IDE，它基于 VS Code 深度定制，并整合了 Antigravity 的 Agent 管理器和任务面板。
  4. 安装完成后，打开 Antigravity IDE，按指引登录 Google 账号并授权访问。

配置并接入 Gemini

  1. 账户登录与激活：首次打开 Antigravity IDE 时，系统会弹出欢迎向导，点击 Login with your Google account。
  2. 注意：登录你拥有 Google One AI Premium（包含 Gemini Advanced 权益）或 Google Cloud / AI Studio 权限的 Google 账号。它会自动识别并激活你的高级 Token 额度。
  3. 如果你是高校用户，可以通过学校的 Google Workspace 教育账户、学院合作协议或校园 Google Cloud 教育计划，申请并免费使用 Google Gemini Pro。许多高校在 Google for Education 或 Cloud 教育计划下会为教师和学生提供免费的 Gemini Pro 试用额度。
  4. 登录成功后，进入 Antigravity IDE 的设置页面，查找 Gemini 或 AI Model 选项，选择 Gemini Advanced / Gemini Pro，并确认模型已激活。
  5. 在需要时，打开 Antigravity 的 Agent Manager，创建新的 Agent 任务并指定目标，如“生成后端登录接口”、“编写前端页面”、“运行自动化测试”等。
Tip：Antigravity 的核心优势在于“由你下高层指令，AI 自动拆解并执行”。使用时，先写清楚目标和验收标准，例如“搭建一个用户登录页面，支持用户名密码登录，数据库存储使用 PostgreSQL，完成后运行接口测试并截图”。
可折叠：Antigravity IDE + Gemini 与 VS Code + Claude 的区别

Antigravity IDE + Gemini 可以作为 VS Code + Claude 的替代方案，但它们的工作方式并不完全相同：

  • 交互逻辑：VS Code + Claude 主要是对话驱动（Chat-driven），你在侧边栏与 AI 互动，AI 帮你写代码或改代码；而 Antigravity IDE + Gemini 是任务驱动（Agent-driven），你给一个最终目标，AI 会在后台自主推理并执行。
  • AI 权限：VS Code + Claude 的权限通常局限于编辑代码文件，绝大多数常规插件默认无法自主运行终端命令或控制浏览器；Antigravity IDE + Gemini 则天然具备系统级权限，能够在中间终端跑 Bash 命令，用 WebFetch 抓网页，自测运行结果，甚至生成截图和录屏。
  • 工作流：VS Code + Claude 更多是同步交互：你写一行，它确认一行，你得一直盯着；Antigravity IDE + Gemini 则是异步并行：你把任务发出去，它可以在后台把整个流水线跑完，最后给你一个结果报告（Artifacts）。

验证 Antigravity 与 Gemini 是否接入成功

在 Antigravity IDE 中创建一个简单任务，例如“使用 Gemini Pro 生成一个 Flask 后端服务，并运行一次本地测试”。如果它能自动创建任务、生成代码、执行服务并给出测试结果，则说明配置成功。

Step 6配置 API 与登录

方式一：使用 Claude 官方账号（推荐）

Claude Code 支持 Claude Pro / Max / Team / Enterprise 订阅账号直接登录，无需单独购买 API 额度。

  1. 在终端或 VS Code 中运行 claude。
  2. 选择登录方式时，选 第1项： 
    ❯ 1. Claude account with subscription · Pro, Max, Team, or Enterprise
  2. Anthropic Console account · API usage billing
  3. 3rd-party platform · Amazon Bedrock, Microsoft Foundry, or Vertex AI
  3. 按提示在浏览器中完成账号授权，返回终端即完成登录。
  4. VS Code 中的 Claude Code 插件会同步可用。
购买渠道：官方订阅在 claude.ai 购买。国内有代购渠道，请注意甄别风险，选择信誉良好的服务商。

方式二：使用国内第三方 API（替代方案）

如果你没有 Claude 官方账号，或想使用国内模型（更稳定、无需代理、价格更低），可以通过配置第三方模型来使用 Claude Code。

两种配置方式：
  • 选项 A：使用 CC Switch（图形化工具，推荐新手） — 图形化工具，下载后填 API Key 即可，无需手写 JSON 配置文件。
  • 选项 B：手动配置（推荐进阶用户） — 直接在 VS Code 的配置文件中设置，灵活度高。

两种方式选其一即可。配置前需要先在模型提供商的官网注册并获取 API Key。

选项 A：使用 CC Switch（图形化工具，推荐新手）

CC Switch 是一种专门用于管理 Claude Code 模型配置的开源图形化工具。它能自动生成并应用配置，避免手动修改 JSON 配置文件的麻烦。macOS 和 Windows 均支持。

使用步骤：
  1. 访问 CC Switch 的官方发布页面（通常在 GitHub Releases 中），下载并安装对应系统的版本：
    • Mac：下载 .dmg 文件，拖入应用程序文件夹。
    • Windows：下载 .exe 安装包，双击进行安装。
    注意：CC Switch 为社区开源工具，请确保下载渠道安全。若有顾虑，请选择选项 B（手动配置）。
  2. 打开 CC Switch，点击 "添加模型" 或 "Add Model"。
  3. 根据提示填写以下信息：
    • API Base URL（API 端点）：填写模型提供商的兼容端点地址（见下表）。
    • API Key：粘贴你获取的 API Key（sk-...）。
    • Model Name（模型名称）：填写对应的模型代号。
  4. 保存并应用，工具会自动将配置同步至 Claude Code，你可以在多个模型间一键切换。
常见兼容端点对照表：
提供商 API Base URL（兼容 Anthropic 协议） 主力模型名称
DeepSeek https://api.deepseek.com/anthropic deepseek-v4-pro
智谱（GLM） https://open.bigmodel.cn/api/paas/v4 glm-4-plus
MiniMax https://api.minimaxi.com/anthropic MiniMax-M2

附录 AStata Workbench 配置

Stata Workbench 插件让你在 VS Code 中直接运行 Stata 代码，并由 Claude Code 辅助调试。

  1. 打开 VS Code Extensions，搜索 "Stata Workbench"，安装。
  2. 成功连接后，右下角状态栏会显示 "Stata Workbench: Connected"。

若未自动连接，需手动配置 Stata 路径。打开 User Settings (JSON)，添加：

{
  "stata-workbench.stataPath": "/Applications/Stata/StataMP.app/Contents/MacOS/StataMP",
  "stata-workbench.mcp.stataPath": "/Applications/Stata/StataMP.app/Contents/MacOS/StataMP"
}
路径以实际安装位置为准。不确定？直接告诉 Claude Code："请帮我配置 Stata Workbench 的路径"，AI 会自动检测。
{
  "stata-workbench.stataPath": "C:\\Program Files\\Stata19\\StataMP-64.exe",
  "stata-workbench.mcp.stataPath": "C:\\Program Files\\Stata19\\StataMP-64.exe"
}
若安装路径不同（如 Stata 18 或 SE 版），相应修改版本号和文件名。

测试

新建文件，将文件语言切换为 Stata（右下角点击"纯文本" → 搜索 Stata），输入：

sysuse auto, clear
summarize

点击运行按钮，若看到统计输出，配置成功。

常见问题

conda activate 报错 "conda: command not found"（Mac） 
source ~/miniforge3/etc/profile.d/conda.sh
conda activate datasci

若要永久修复，将上面第一行添加到 ~/.zshrc。

VS Code 找不到 Python 解释器

按 Cmd/Ctrl+Shift+P → 输入 "Python: Select Interpreter" → 选择 datasci (Python 3.10.x)。

claude 命令运行后一直转圈或显示连接错误

可能的原因和解决方法：

  1. 代理端口不对：确认 VPN 客户端的本地监听端口（常见：7890、1080、8080），修改代理设置中的端口号再试。
  2. VPN 未开启或未连接：确保代理工具已启动并正常工作。
  3. API Key 无效：如果使用第三方 API，检查 API Key 是否正确、是否有余额。
安装 Claude Code 时报 "EACCES: permission denied"

Mac：

方法一：在命令前加 sudo：

sudo npm install -g @anthropic-ai/claude-code

方法二：更改 npm 全局目录到用户目录（推荐，避免权限问题）：

mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc
source ~/.zshrc
npm install -g @anthropic-ai/claude-code

Windows：以管理员身份运行 PowerShell，再执行 npm install -g。

Mac 上提示 "claude: command not found"

通常是 npm 全局 bin 目录不在 PATH 中。运行以下命令修复：

echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc
source ~/.zshrc

如果是用 sudo 安装的，检查 /usr/local/bin 是否在 PATH 中：

echo $PATH
VS Code 中 Claude Code 插件不显示或无法使用

请依次检查：

  1. 确认 Claude Code CLI 已正确安装：终端/PowerShell 运行 claude --version 看是否有输出。
  2. 确认 VS Code 插件已安装：Extensions 中搜索 "Claude Code"，确保已启用（不要处于禁用状态）。
  3. 尝试完全关闭 VS Code 再重新打开（Mac: Cmd+Q / Windows: 右键任务栏图标退出）。
  4. 如果配置了第三方 API，确认 settings.json 中 claudeCode.disableLoginPrompt 设为 true。
配置了第三方 API，但 Claude Code 仍然提示需要登录

在 VS Code 的 settings.json 中确认已添加：

"claudeCode.disableLoginPrompt": true

然后完全关闭 VS Code 再重新打开（不重启配置不生效）。

如果使用命令行运行 claude，可通过环境变量跳过：

Mac：

export CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1
claude

Windows：

$env:CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC="1"
claude
API Key 如何保管？会不会泄露？

API Key 存储在 VS Code 的 settings.json 中，仅在本地有效。注意：

  • 不要将包含 API Key 的配置文件上传到 GitHub 等公开平台。
  • 如果不小心泄露，立即去对应平台删除旧 Key 并创建新 Key。
  • 大部分平台支持设置 API Key 的用量上限，建议设置一个合理的月度上限以防意外。
VS Code 配置文件有多个大括号，如何合并？

User Settings (JSON) 文件只能有一个顶层 {} 对象。若已有其他配置，将新增字段并入同一个 {} 内，用逗号分隔，不要出现两个独立的 {...}。

例如，合并前（错误）：

{
  "editor.fontSize": 14
}
{
  "claudeCode.environmentVariables": [...]
}

合并后（正确）：

{
  "editor.fontSize": 14,
  "claudeCode.environmentVariables": [...]
}

Claude 官方国内模型
